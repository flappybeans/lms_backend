BORROW BOOK
ENTER BORROWER AND BORROW TRANSACTION DETAILS
CONFIRM
STORE IN BORROWEDBOOK TABLE
BORROWER GETS LISTED IN QUEUE(HE IS CURRENTLY THE BORROWER)
UPDATES BOOK TABLE
BOOK GETS NEW BORROWER
NEW BORROWER GETS LISTED IN THE QUEUE NEXT TO THE CURRENT BORROWER
CURRENT BORROWER RETURNS THE BOOK
CURRENT BORROWER GETS REMOVED FROM THE QUEUE
THE BORROW QUEUE GOT UPDATED
THE BOOK STATUS GOT UPDATED TO WAITING TO BE CLAIMED
IF BOOK NOT CLAIMED WITHIN THE EXPIRY DATE
    THEN THE BORROWER TO CLAIM THAT BOOK GOT REMOVED IN THE LIST
    THEN THE NEXT BORROWER AFTER THAT BORROWER IS NOW THE CURRENT BORROWER WAITING TO CLAIM THE BOOK
ELSE
    MARK THE BOOK AS CLAIMED
    UPDATE THE BOOK TABLE
    UPDATE THE BORROWEDBOOK TABLE
